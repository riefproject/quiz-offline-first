# Spesifikasi Kebutuhan Sistem: AlpenQuiz

---

## 1. User Requirements (UR)

Dokumen ini merangkum spesifikasi kebutuhan pengguna (_User Requirements_) untuk sistem kuis berbasis luring (_offline-first_). Spesifikasi ini berfokus pada objektif operasional dan ekspektasi fungsional dari sudut pandang pemangku kepentingan (_stakeholders_).

| Kode Requirement | Stakeholder | Deskripsi |
| :--------------- | :---------- | :-------- |
| **UR-01** | Peserta | Pengguna dapat mengakses dan menyelesaikan sesi kuis tanpa ketergantungan pada konektivitas internet eksternal. |
| **UR-02** | Host | Pengguna diwajibkan mendaftarkan akun secara resmi untuk mendapatkan hak istimewa (_privilege_) penyelenggaraan dan pengelolaan kuis. |
| **UR-03** | Peserta | Pengguna dapat memantau perolehan skor kuis secara seketika (_real-time_) sesaat setelah sesi kuis dinyatakan selesai. |
| **UR-04** | Host | Pengguna memiliki kapabilitas untuk melakukan manajemen paket kuis (pembuatan, penyuntingan, dan peluncuran) secara mandiri melalui perangkat seluler. |
| **UR-05** | Host | Pengguna memiliki jaminan integritas data peserta melalui mekanisme sinkronisasi otomatis ke peladen pusat ketika jaringan internet kembali tersedia. |
| **UR-06** | Host | Pengguna dapat menyelenggarakan dan mendistribusikan sesi kuis secara terpusat melalui jaringan _Local Area Network_ (LAN) dalam kondisi infrastruktur internet eksternal terputus. |
| **UR-07** | Peserta | Pengguna dapat bergabung dan mengikuti sesi kuis secara anonim (_guest mode_) tanpa perlu melewati hambatan proses pendaftaran akun. |
